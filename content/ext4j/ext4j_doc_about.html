Why Ext4j ?

When it comes to Rich Internet Applications Sencha's Ext JS is definitely one the richest and most powerful library on the market. Not only does Ext JS help developers build amazing web based applications, the community around Ext JS is one the most active and most creative.

At NTT DATA we have been writing big scale J2EE applications with Ext JS as the framework of choice for the UI for years now. Even though we were always able to get things done(and believe us we wrote some of the craziest Ext JS based applications), there is one thing we could'nt deny : As the code base grows JavaScript becomes extremely difficult to maintain and work with. And when the team is spread around the globe the maintainability and scalability problem becomes even bigger.

This is not an attempt to diss the JavaScript language! With the browser becoming a stronger platform everyday we all know that there is no doubt that JavaScript will play an even bigger role in the future. But there are facts that can't by denied and made us think about a new way of writing enterprise applications with Ext:

- Compile time errors
Being able to catch errors before the application even runs is capital for any project(specially big scale one).

- Tooling a.k.a IDE Support
JavaScript tooling has gotten better and better over the time. IDEs like Aptana and JetBrain's IntelliJ have made writing JavaScript code enjoyable. Sencha just released a set of tools to make Ext JS development easier. But real software development goes beyond autocompletion and content assist. Refactoring JavaScript code for example is not easy and a JS based tool will only be able to do basic string search and replace.

- Real Software Engineering
Interface based design, Design patterns, Composition, Encapsulation, this are things that are not built in the JavaScript language. Off course they can be emulated and Sencha came up with a nice class System for Ext JS. The problem with emulation is that there are tons of way to do the same things. While working on a big scale project it s almost impossible to ensure that everyone is following the same code style.

- Backend Integration
Even though Ext JS provides some nice tools to deal with any type of backend (Ext Direct) one always has to deal with two programming models(unless she is working with NodeJS). The ability to leverage to same language through out the entire project is a huge productivity boost.

Meet Ext4j

The Extj library is our attempt to solve the problems mentioned above by giving developers the ability to program in Java and having the Google Web Toolkit Compiler generate high efficient JavaScript for you.

Note that this is not about promoting the Java language or stating that Java is a better language than JavaScript. But Java was designed from the ground up to support big scale applications. Plus the tons of free available powerful tools in the Java ecosystem makes it a good candidate for enterprise software.

This toolkit targets project that must use Ext JS but not necessary JavaScript as the input language. If your project is not bound the an Ext JS license(like ours) and you are free to choose another library you might want to consider Sencha's GXT library.

In our case our customers allready have an Ext JS license an were not willing to buy a license for a new product(Yes Ext JS and GXT require two different licenses).

Now what does Ext4j has to offer ? let s look at some of the features

- Free to use
Ex4j is free to use with an Apache 2 license.So you dont owe NTT Data anything to use this library. But keep in mind that EXT JS is not free. You will need a commercial license if you are building a commercial application.

- 100% implementation of Ext JS API
Starting with EXT JS 4.1 Ext4j aims to support all the features of Ext and add even more capabilities that native JavaScript development can not provide

- Strong tooling support
This is pretty obvious. Because we leverage Java you get all the goodies of Java development and that for free!.Lets have a look a some of the features available.

*) Code completion: Explore the methods and fields available to the object you're working on.



*) Outline view: List the classes, methods, and functions in a simple tree display.



*) Debugging: Set breakpoints, inspect variables, and step over, into, and out of code..



*)Static Analysis: See warnings when inconsistencies and potential problems are detected..



- Easy to learn
The API is simple and aims look familiar to all developers.

- Declarative User Interface (a.k.a UI Binders)
Let's face Java is pretty verbose and one can end up writing a lot of code to layout the UI. That s why Ext4j introduced support for GWT UI Binders. UI binders give the ability the layout the UI in a more declarative way.
Think about how you would write the same code in Java or in JavaScrit. Plus a designer will understand this code better than the Java or JavaScript version of it.

- Easy (JEE) backend Integration
Because Ext4j leverage to Google Web Toolkit. Integrating any type of backend is a piece of cake. But if your backend is Java then you will love Ext4j. GWT provides mechanisms to directly call Java methods from within you client side code. No need of Ext Direct or anything like that.

- A lot more ...
We just scratched the surface here. Ext4j offers a lot more feature and we do have lot in the pipeline for the coming releases. To see what s possible with Ext4j have a look at the - examples section.
