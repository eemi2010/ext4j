Why a new approach on Ext JS Development ?

When it comes to Rich Internet Applications Sencha's Ext JS is definitely one the richest and most powerful library on the market.

At NTT DATA we have been writing big scale J2EE applications with EXT JS as the framework of choice for the UI for years now. Even though we were always able to get things done(and believe us we wrote some of the craziest Ext JS based applications), there is one thing we could'nt deny : JavaScript is a nightmare when it comes to big scale applications!.

This is not an attempt to diss JavaScript! With the browser becoming a stronger platform everyday we all know that there is no way around JavaScript for browser based applications. But there are facts that can't by denied and made us think about a new way of writing enterprise applications with EXT JS:

- Compile time errors
Being able to catch errors before the application even runs is capital for any project(specially big scale one).

- Tooling a.k.a IDE Support
JavaScript tooling has gotten better and better over the time. IDEs like Aptana and JetBrain have made writing JavaScript code enjoyable. Sencha just released a set of tools to make EXT JS development easier. But real software development goes beyond autocompletion and content assist. Refactoring JavaScript code for example is not easy and no JS tool will help you with that.

- Real Software Engineering
Interface based design, Design patterns, Composition, Encapsulation, this are things that are not built in the JavaScript language and need to be emulated. The problem with emulation is that there are tons of way to do the same things. While working on a big scale project it s almost impossible to ensure that everyone is following the same code style.

- Backend Integration
Even though Ext JS provides nice tools to deal with any type of backend (Ext Direct) we always have to deal with two programming models because of backend ist mostly based on Java. The ability to leverage to same language through out the entire project is a huge productivity bost.

Meet Ext4j

The Extj library is an attempt to solve the problems mentioned above by giving developers the ability to program in Java and having the Google Web Toolkit generate high efficient JavaScript for you.

Note that this is not about promoting the Java language or saying that Java is better than JavaScript. But Java was designed from the ground up to support big scale applications. Plus the megaton of free available tools in the Java ecosystem makes it a good candidate for enterprise software.

This toolkit targets project that must use EXT JS but not necessary JavaScript as the input language. If your project is not bound the an EXT JS license(like ours) and you are free to choose another library you might want to consider Sencha's GXT library.

In our case our customers allready have an Ext JS license an dare not willing to buy a license for a new product(Yes EXT JS and GXT are two different products. Even if they come from the same company).

Now what does Ext4j has to offer ? let s look at some of the features

- 100% implementation of EXT JS API

Starting with EXT JS 4.1 Ext4j aims to support all the features of EXT and add even more stuff that native JavaScript development can not provide - Strong typing and compile time errors

This is pretty obvious. Because we leverage Java you get all the goodies of Java development and that for free! The API is simple and easy to read for any one.

- Declarative User Interface (a.k.a UI Binders)

Let's face Java is pretty verbose and one can end up writing a lot of code to layout the UI. That s why Ext4j introduced support for GWT UI Binders. UI binders give the ability the layout the UI in a more declarative way.
Think about how you would write the same code in Java or in JavaScrit. Plus a designer will understand this code better than the Java or JavaScript version of it.

- Easy (JEE) backend Integration

Because Ext4j leverage to Google Web Toolkit. Integrating any type of backend is a piece of cake. But if your backend is Java then you will love Ext4j. GWT provides machanisms to directly call Java methods frm within you client side code. No need of Ext Direct or anything like that.

- Free to use

Ex4j is free to use with an Apache 2 license. Keep in mind that EXT JS is not free though. You will need a commercial license if you are building a commercial application.
